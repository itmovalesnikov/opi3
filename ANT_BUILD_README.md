# Apache Ant Build Script для Web3 Проекта

## Описание

Создан полный сценарий Apache Ant (`build.xml`) для компиляции, тестирования и упаковки проекта Web3 из лабораторной работы №3 по веб-программированию.

## Структура сценария

### Файлы конфигурации

1. **`build.properties`** - Файл параметров, содержит:
   - Параметры проекта (название, версия, автор)
   - Пути исходных кодов и ресурсов
   - Пути сборки и дистрибутива
   - Параметры компиляции (версия Java, кодировка)
   - Параметры документации и архивов

2. **`build.xml`** - Главный файл сценария со всеми целями

## Реализованные цели (Targets)

### 1. `compile` - Компиляция исходных кодов
```bash
ant compile
```
Осуществляет:
- Компиляцию всех Java файлов из `src/main/java` в выходной каталог `build/classes`
- Копирование ресурсов из `src/main/resources`
- Использование параметров: Java 17, UTF-8 кодировка, debug информация

**Этап пути выполнения:** Зависит от цели `init`

### 2. `build` - Компиляция и упаковка в JAR
```bash
ant build
```
Осуществляет:
- Вызов цели `compile` для компиляции исходных кодов
- Создание MANIFEST.MF файла с метаинформацией:
  - Версия проекта
  - Названием реализации
  - Информацией о разработчике
  - Датой и временем сборки
  - Версией Java для компиляции
- Создание JAR архива `dist/web3.jar` со всеми скомпилированными классами и ресурсами
- Вывод размера созданного архива

**Этап пути выполнения:** Зависит от цели `compile` → `init`

### 3. `clean` - Очистка временных файлов
```bash
ant clean
```
Удаляет:
- Весь каталог `build/` (все скомпилированные классы)
- Весь каталог `dist/` (все созданные архивы)
- Резервные файлы (`*.bak`, `*~`)

**Этап пути выполнения:** Независимая цель

### 4. `test` - Запуск JUnit тестов
```bash
ant test
```
Осуществляет:
- Сборку проекта перед запуском тестов (вызов цели `build`)
- Проверку наличия тестов в `src/test/java`
- Компиляцию тестовых классов в `build/test-classes`
- Запуск всех классов, заканчивающихся на `Test` и `Tests`
- Сохранение результатов в формате XML и plain text в `build/reports/`

**Этап пути выполнения:** Зависит от цели `build` → `compile` → `init`

### 5. `doc` - Генерация Javadoc и контрольных сумм
```bash
ant doc
```
Осуществляет:
- Генерацию полной Javadoc документации для всех классов
- Расчёт контрольных сумм JAR архива:
  - MD5 хеш файла
  - SHA-1 хеш файла
- Обновление MANIFEST.MF файла с добавлением контрольных сумм
  - `X-SHA1-Checksum`
  - `X-MD5-Checksum`
- Пересоздание JAR архива с обновленным манифестом
- Добавление Javadoc документации внутрь JAR архива с префиксом `docs/javadoc`

**Этап пути выполнения:** Зависит от цели `build` → `compile` → `init`

### 6. `team` - Сборка предыдущих git ревизий
```bash
ant team
```
Осуществляет:
- Проверку наличия git репозитория (`.git` директория)
- Получение 2 последних коммитов до текущего
- Для каждой предыдущей ревизии:
  - Клонирование репозитория в временный каталог
  - Переключение на требуемый коммит
  - Запуск цели `build` для этой ревизии
  - Создание отдельного JAR архива (`web3-rev1.jar`, `web3-rev2.jar`)
- Сбор всех JAR архивов (текущей версии и 2 предыдущих) в ZIP архив `dist/team-builds.zip`

**Этап пути выполнения:** Зависит от цели `build` → `compile` → `init`

## Вспомогательные цели

### `help` - Справка
```bash
ant help
```
Выводит список всех доступных целей и примеры их использования

### `info` - Информация о проекте
```bash
ant info
```
Выводит подробную информацию о параметрах проекта и путях каталогов

## MANIFEST.MF структура

### Базовый MANIFEST.MF (для целей compile/build):
```
Manifest-Version: 1.0
Implementation-Title: web3
Implementation-Version: 1.0-SNAPSHOT
Implementation-Vendor: ITMO Lab3
Created-By: Apache Ant
Build-Date: YYYY-MM-DD HH:MM:SS
Build-Java-Version: X.X.X
X-Project-Encoding: UTF-8
```

### Расширенный MANIFEST.MF (после цели doc):
```
<все базовые атрибуты>
X-SHA1-Checksum: <SHA-1 хеш>
X-MD5-Checksum: <MD5 хеш>
```

## Структура файлов и каталогов

```
opi3/
├── src/
│   ├── main/
│   │   ├── java/                    # Исходные Java коды
│   │   ├── resources/               # Ресурсы (файлы конфигурации)
│   │   └── webapp/                  # Веб ресурсы
│   └── test/
│       ├── java/                    # Тесты (JUnit)
│       └── resources/               # Тестовые ресурсы
├── lib/
│   ├── compile/                     # Библиотеки для компиляции
│   ├── package/                     # Библиотеки для упаковки (внутрь JAR)
│   └── test/                        # Библиотеки для тестирования
├── build/                           # Директория сборки (создаётся автоматически)
│   ├── classes/                     # Скомпилированные классы
│   ├── test-classes/                # Скомпилированные тесты
│   ├── docs/javadoc/                # Сгенерированная Javadoc
│   ├── reports/                     # Результаты тестов
│   ├── temp/                        # Временные файлы
│   └── META-INF/                    # Манифест проекта
├── dist/                            # Директория дистрибутива (создаётся автоматически)
│   ├── web3.jar                     # Созданный JAR архив
│   └── team-builds.zip              # ZIP архив с предыдущими версиями
├── build.xml                        # Сценарий Apache Ant
├── build.properties                 # Параметры сборки
└── pom.xml                          # Конфигурация Maven (для справки)
```

## Примеры использования

### 1. Базовая компиляция и сборка
```bash
ant build
```

### 2. Полная сборка с документацией
```bash
ant clean build doc
```

### 3. Сборка и запуск тестов
```bash
ant clean build test
```

### 4. Сборка со всеми возможностями
```bash
ant clean build test doc team
```

### 5. Очистка проекта
```bash
ant clean
```

### 6. Сборка с пользовательскими параметрами
```bash
ant build -Djar.filename=custom-name.jar -Dproject.version=2.0
```

## Параметры в build.properties

Параметры можно переопределить в командной строке с помощью флага `-D`:

```bash
ant build \
  -Dproject.version=1.1-SNAPSHOT \
  -Djar.filename=web3-v1.1.jar \
  -Djava.source.version=16 \
  -Djava.target.version=16
```

## Требования

- **Apache Ant** версии 1.10+ 
  - Установка: `sudo apt install ant` (Linux/Debian)
  - Или скачать с https://ant.apache.org/
- **Java SDK** 17+
  - Для компиляции и запуска
- **Git** (опционально, для цели `team`)
  - Требуется для получения предыдущих ревизий

## Особенности реализации

### Разделение на этапы
- **Этап 1 (compile):** Компиляция исходных кодов
- **Этап 2 (build):** Создание JAR архива
- **Этап 3 (clean):** Очистка файлов
- **Этап 4 (test):** Запуск тестов
- **Этап 5 (doc):** Генерация документации
- **Этап 6 (team):** Обработка git ревизий

### Использование параметров
- Все константы вынесены в `build.properties`
- Легко настроить под конкретный проект
- Параметры можно переопределять из командной строки

### MANIFEST.MF информация
- Содержит информацию о версии и запускаемом классе (хотя JAR не исполняемый, это веб-приложение)
- Содержит контрольные суммы SHA-1 и MD5 после цели `doc`
- Содержит информацию о разработчике и дате сборки

### Обработка Git ревизий
- Автоматически получает 2 последних коммита
- Клонирует репозиторий для каждой ревизии
- Собирает каждую версию отдельно
- Упаковывает все JAR файлы в один ZIP архив

## Примечания

1. Проект является веб-приложением (JAR используется в качестве архива классов, а не исполняемого приложения)
2. Для работы цели `team` проект должен находиться в git репозитории
3. Для запуска тестов необходимо наличие JUnit библиотек в `lib/test/`
4. Javadoc генерируется для пакета `com.github.itmovalesnikov.web3.*`
